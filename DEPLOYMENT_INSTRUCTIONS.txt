BIO CELLR8 LLC WEBSITE - DEPLOYMENT INSTRUCTIONS
================================================

This document provides step-by-step guidance for deploying the BioCellR8 professional website to GitHub Pages.

PREREQUISITES
-------------
1. Git installed on your computer (https://git-scm.com/downloads)
2. A GitHub account with push access to: https://github.com/pixelpolicyuser/biocellr8
3. Basic familiarity with file management (copy/paste folders)
4. (Optional but recommended) GitHub Desktop for a graphical interface

STEP-BY-STEP DEPLOYMENT
-----------------------

Step 1: Prepare Your Local Copy
- Locate the folder: biocellr8-website (contains index.html, css/, js/, etc.)
- Make sure all your final edits are complete and tested locally by opening index.html in a browser.

Step 2: Initialize Git (First Time Only)
If this is your first deployment:
   Open Terminal / Command Prompt / PowerShell in the biocellr8-website folder and run:
   
   git init
   git remote add origin https://github.com/pixelpolicyuser/biocellr8.git
   git branch -M main

Step 3: Add and Commit Your Files
   git add .
   git commit -m "Initial professional website deployment for BioCellR8 LLC"

Step 4: Push to GitHub
   git push -u origin main

Step 5: Enable GitHub Pages (One-time setup in GitHub)
   1. Go to https://github.com/pixelpolicyuser/biocellr8
   2. Click "Settings" → "Pages"
   3. Under "Source", select **"GitHub Actions"** (recommended for the automated workflow)
   4. Click "Save"
   5. The first push to `main` will trigger the deployment workflow automatically.
   6. Wait 1-2 minutes after the Actions run completes — your site will be live at: https://pixelpolicyuser.github.io/biocellr8/

FUTURE UPDATES (After First Deployment)
---------------------------------------
**Recommended: Automatic Deployment via GitHub Actions**
- Simply push your changes to the `main` branch:
  ```bash
  git add .
  git commit -m "Update: Brief description of changes"
  git push origin main
  ```
- A GitHub Actions workflow will automatically:
  1. Detect the push
  2. Build and validate the site
  3. Deploy to GitHub Pages
- Monitor progress in the **Actions** tab of the repository.
- The live site updates in ~30-90 seconds.

**Local Convenience (Optional)**
- Use the included PowerShell script for quick local commits:
  1. Open PowerShell in the biocellr8-website folder
  2. Run: `.\deploy.ps1`
  3. The script stages, commits with a professional message, and pushes — triggering the same auto-deploy workflow.

**Manual Git (Fallback)**
- Same commands as above; GitHub Actions handles the rest.

TROUBLESHOOTING
---------------
- "Permission denied" or authentication errors:
  → Use GitHub Desktop or configure a Personal Access Token (PAT) in Git.
  
- Changes not appearing on the live site:
  → Wait 60-90 seconds after the GitHub Actions workflow completes.
  → Hard-refresh the browser (Ctrl + Shift + R).
  → Go to the **Actions** tab → click the latest "Deploy BioCellR8 Website to GitHub Pages" run for detailed logs and deployment URL.

- "404.html" or broken links:
  → All links in the site are relative. Ensure you did not move files outside the root folder.

- Want to use a custom domain later?
  → Add a CNAME file in the root with your domain name.
